## PCHelper — Phase 11a: Performance Monitor Overlay (性能监视器浮窗)

### Overview
Add a floating performance monitor overlay (HUD) that shows real-time CPU, Memory, GPU, and FPS stats on top of all windows. Think MSI Afterburner RivaTuner style — compact, semi-transparent, always-on-top overlay.

### Architecture

```
electron/
├── overlay/
│   ├── overlayManager.ts    — Manages the translucent always-on-top overlay window
│   └── dataCollector.ts     — Collects hardware metrics at configurable intervals

src/
├── components/
│   ├── PerfOverlay/
│   │   ├── index.tsx        — Overlay editor/dashboard (main page)
│   │   ├── OverlayPreview.tsx  — Live preview of what the overlay looks like
│   │   └── SettingsForm.tsx    — Toggle metrics, colors, position, opacity
├── stores/
│   └── overlayStore.ts      — Zustand state for overlay settings
```

### Electron Overlay Window (`electron/overlay/overlayManager.ts`)

Create a **secondary Electron BrowserWindow** that:
- Is frameless, always-on-top, click-through (transparent mouse events)
- Has a transparent background
- Position: user-configurable (default: top-right)
- Size: ~280x140 pixels
- Opacity: adjustable (default 70%)
- Updates via IPC from the main renderer

```typescript
export interface OverlayConfig {
  enabled: boolean;
  position: 'top-right' | 'top-left' | 'bottom-right' | 'bottom-left';
  opacity: number; // 0.3 - 1.0
  metrics: {
    showCpu: boolean;
    showMemory: boolean;
    showGpu: boolean;
    showFps: boolean;
    showNetwork: boolean;
  };
  fontSize: number;
  refreshInterval: number; // ms, default 1000
  clickThrough: boolean;
  autoHide: boolean; // hide when no fullscreen app running
  accentColor: string;
}
```

Key methods:
- `toggleOverlay(enabled: boolean)` — Create or destroy overlay window
- `updateOverlayConfig(config: OverlayConfig)` — Update position/opacity/metrics
- `sendMetrics(data: OverlayMetrics)` — Push data to overlay window (via IPC)
- `destroy()` — Clean up

The overlay window loads a simple HTML page (built with Vite, separate entry point) — but for simplicity, embed a minimal HTML template via `loadURL('data:text/html,...')` or a simple React component that gets bundled.

**Actually, simpler approach:** Create a dedicated HTML page at `electron/overlay/overlay.html` that uses vanilla JS + CSS. It receives data via IPC and renders it. No React overhead for the overlay.

### Frontend (`src/components/PerfOverlay/`)

**Main Page (`index.tsx`):**
- On/off toggle for the overlay
- Live preview thumbnail showing what the overlay looks like
- Quick-access toggle buttons: "Toggle Overlay" / "Toggle Click-Through"
- FPS counter source selector: "Auto" / "None (no FPS)"
- "Start on Windows startup" checkbox

**Preview (`OverlayPreview.tsx`):**
- A small box that mimics the overlay appearance
- Shows simulated data with the current theme colors
- Draggable to preview positioning

**Settings (`SettingsForm.tsx`):**
- Position: dropdown (top-right/default/top-left/bottom-right/bottom-left)
- Opacity: slider 30-100%
- Font size: small/medium/large
- Refresh rate: 250ms / 500ms / 1s / 2s / 5s
- Metrics to display: checkboxes for CPU/Mem/GPU/FPS/Network
- Accent color: color picker
- Click-through toggle
- Auto-hide in fullscreen toggle

### IPC Handlers

```
pchelper:overlay-toggle       -> toggleOverlay(enabled: boolean)
pchelper:overlay-update-config -> updateOverlayConfig(config: OverlayConfig)
pchelper:overlay-get-status    -> returns { active: boolean, config: OverlayConfig }
pchelper:overlay-get-metrics   -> returns current OverlayMetrics snapshot
```

### Preload Bridge

```
togglePerfOverlay(enabled: boolean)

getOverlayStatus(): Promise<{ active: boolean, config: OverlayConfig }>
```

4. Keep the dark theme consistent, use existing common components
