## Phase C2: 桌面小组件系统

项目：PCHelper (pc-toolkit) — Windows 系统工具，Electron + React + Vite + TypeScript + Zustand + better-sqlite3 + Recharts + lucide-react

### Task

在 Dashboard 中实现可拖拽、可配置的桌面小组件系统。每个面板（CPU、内存、磁盘、GPU、健康分、温度概览、内存构成、磁盘活动、性能历史）都可以作为独立小组件，用户可自由拖拽排序、调整大小、显示/隐藏。

#### 2. `electron/widget/widgetManager.ts` — SQLite CRUD
```typescript
import Database from 'better-sqlite3';
import path from 'path';
import { app } from 'electron';
import { WidgetLayout, DEFAULT_WIDGETS } from './types';

const dbPath = path.join(app.getPath('userData'), 'widget.db');

let db: Database.Database | null = null;

function getDb(): Database.Database {
  if (!db) {
    db = new Database(dbPath);
    db.pragma('journal_mode = WAL');
    db.exec(`CREATE TABLE IF NOT EXISTS widget_layout (id INTEGER PRIMARY KEY CHECK (id = 1), config TEXT NOT NULL)`);
  }
  return db;
}

export function getWidgetLayout(): WidgetLayout {
  const d = getDb().prepare('SELECT config FROM widget_layout WHERE id = 1').get() as { config: string } | undefined;
  if (d) return JSON.parse(d.config);
  return { widgets: DEFAULT_WIDGETS };
}

export function saveWidgetLayout(layout: WidgetLayout): void {
  getDb().prepare('INSERT INTO widget_layout (id, config) VALUES (1, ?) ON CONFLICT(id) DO UPDATE SET config = excluded.config').run(JSON.stringify(layout));
}

export function getWidgetDefaults(): WidgetLayout {
  return { widgets: DEFAULT_WIDGETS };
}
```

#### 4. `src/stores/widgetStore.ts` — Zustand store
```typescript
import { create } from 'zustand';
import type { WidgetConfig, WidgetId, WidgetLayout } from '../../electron/widget/types';

interface WidgetState {
  layout: WidgetLayout | null;
  loading: boolean;
  dragWidgetId: WidgetId | null;

  loadLayout: () => Promise<void>;
  updateWidget: (id: WidgetId, partial: Partial<WidgetConfig>) => Promise<void>;
  reorderWidget: (fromIndex: number, toIndex: number) => Promise<void>;
  resetLayout: () => Promise<void>;
  setDragWidget: (id: WidgetId | null) => void;
}

export const useWidgetStore = create<WidgetState>((set, get) => ({
  layout: null,
  loading: true,
  dragWidgetId: null,

  loadLayout: async () => {
    try {
      const layout: WidgetLayout = await window.pchelper.getWidgetLayout();
      set({ layout, loading: false });
    } catch {
      set({ loading: false });
    }
  },

  updateWidget: async (id, partial) => {
    const current = get().layout;
    if (!current) return;
    const widgets = current.widgets.map((w) => w.id === id ? { ...w, ...partial } : w);
    const next = { widgets };
    await window.pchelper.saveWidgetLayout(next);
    set({ layout: next });
  },

  reorderWidget: async (fromIndex, toIndex) => {
    const current = get().layout;
    if (!current) return;
    const widgets = [...current.widgets];
    const [moved] = widgets.splice(fromIndex, 1);
    widgets.splice(toIndex, 0, moved);
    const reordered = widgets.map((w, i) => ({ ...w, order: i }));
    const next = { widgets: reordered };
    await window.pchelper.saveWidgetLayout(next);
    set({ layout: next });
  },

  resetLayout: async () => {
    const defaults: WidgetLayout = await window.pchelper.getWidgetDefaults();
    await window.pchelper.saveWidgetLayout(defaults);
    set({ layout: defaults });
  },

  setDragWidget: (id) => set({ dragWidgetId: id }),
}));
```

#### 5. `src/components/Dashboard/WidgetGrid.tsx` — 可拖拽小组件网格容器
一个可拖拽排序的小组件布局容器：

- **接收 `widgets` 数组**（按 order 排序、过滤 visible===true）
- **每个 widget 渲染为包裹 div**，带拖拽手柄和尺寸控制
- **拖拽机制**：用原生 HTML5 Drag & Drop（不引入外部库）
  - 每个 widget 的 header 部分作为 drag handle
  - `draggable`, `onDragStart`, `onDragOver`, `onDrop`, `onDragEnd`
  - 拖拽过程中添加视觉反馈（半透明 placeholder、高亮目标位置）
- **布局计算**：将宽屏设置为 grid，每个 widget 根据 `width` 属性占据相应列数
  - 每行最多 4 列
  - `width=1` 占 1/4，`width=2` 占 2/4，`width=3` 占 3/4，`width=4` 占整行
  - 使用 `grid-column: span N` 实现
- **height 控制**：compact 模式给更小的 min-height
- **widget 头部**：左侧拖拽手柄（GripVertical 图标）、中间标题、右侧折叠按钮（ChevronDown/Up）
- **空状态**：所有 widget 隐藏时显示 "No widgets visible. Open Widget Panel to add some."
- **主题联动**：读取 `useThemeStore` 的 cssVars

CSS：`src/components/Dashboard/WidgetGrid.css`

#### 4. `src/components/Dashboard/index.tsx` — 重构为 WidgetGrid

**大重构思路：**
1. 删除 `orderedCards` 中的 5 个 StatCard + 健康分卡片
2. 删除 `TempCard` 三列温度行
3. 删除 `memComposition` + `diskActivity` 两部分
4. 删除性能历史 tab 区域
5. 删除 Memory Details + Storage 两部分
6. 删除 `themePanelOpen` 相关按钮
7. **改为渲染 `WidgetGrid`**，将每个面板转为独立的 widget 子组件

**具体修改：**

1. 将原有的所有仪表盘区域拆分为独立小组件函数，放在 Dashboard 组件**上方**（或同一文件内）：
   - `CpuWidget` — 原 CPU StatCard + 展开详情区域
   - `MemoryWidget` — 原 Memory StatCard
   - `DiskWidget` — 原 Disk StatCard
   - `GpuWidget` — 原 GPU StatCard
   - `HealthWidget` — 原健康分圆形卡片
   - `TemperatureWidget` — 原三列温度卡片（CPU/GPU/Disk）
   - `MemCompositionWidget` — 原 Memory Composition 堆叠面积图 + 图例
   - `DiskActivityWidget` — 原 Disk Activity 区域
   - `PerfHistoryWidget` — 原 Performance History tabs + 面积图
   - `MemoryDetailsWidget` — 原 Memory Details 面板
   - `StorageWidget` — 原 Storage 面板

   每个 widget 组件接收 `snapshot`、`history`、`healthScore` 等原有 props，渲染为纯粹的展示内容（套在 WidgetGrid widget-item 的 body 中）。
