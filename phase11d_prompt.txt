## PCHelper — Phase 11d: System Cleanup & Optimization (系统优化清理)

### Overview
Add a system cleanup tool with temp file cleaning, browser cache cleanup, Windows update cache cleanup, large file scanner, and duplicate file finder.

### Architecture

```
electron/
├── cleanup/
│   ├── tempCleaner.ts     — Temp file scanner + cleaner
│   ├── browserCleaner.ts  — Browser cache manager
│   ├── systemCleaner.ts   — Windows update cache, logs, prefetch
│   ├── largeFileScanner.ts — Large file finder
│   └── duplicateFinder.ts  — Duplicate file detector (by name+size+hash)

src/
├── components/
│   └── CleanupView/
│       ├── index.tsx        — Main view with 3 tabs: Quick Clean, Large Files, Duplicates
│       ├── QuickClean.tsx   — Temp/browser/system cleanup with checklist
│       ├── LargeFiles.tsx   — Large file scanner with results table
│       └── DuplicateFiles.tsx — Duplicate file finder with results
├── stores/
│   └── cleanupStore.ts   — Zustand state
```

### Backend (`electron/cleanup/`)

#### `tempCleaner.ts`
```typescript
export interface TempFileCategory {
  id: string;
  name: string;
  path: string;
  description: string;
  size: number; // bytes
  fileCount: number;
  canClean: boolean;
}

export interface CleanupResult {
  category: string;
  filesDeleted: number;
  spaceFreed: number; // bytes
  errors: string[];
}

export class TempCleaner {
  scanTempFiles(): TempFileCategory[]
  cleanTempFiles(categories: string[]): Promise<CleanupResult[]>
}
```

Categories: Windows Temp (`C:\Windows\Temp`), User Temp (`AppData\Local\Temp`), Prefetch, Recycle Bin, Recent Documents, Delivery Optimization Files.

#### `browserCleaner.ts`
```typescript
export interface BrowserCacheInfo {
  browser: string; // Chrome, Edge, Firefox, etc.
  cacheSize: number;
  cookieCount: number;
  historySize: number;
  lastCleared: string;
}

export class BrowserCleaner {
  scanBrowserCaches(): BrowserCacheInfo[]
  cleanBrowserCache(browser: string): Promise<{ size: number; message: string }>
}
```

#### `systemCleaner.ts`
```typescript
export interface SystemCleanupCategory {
  id: string;
  name: string;
  description: string;
  size: number;
  canClean: boolean;
  risk: 'safe' | 'moderate' | 'warning';
}

export class SystemCleaner {
  scanSystem(): SystemCleanupCategory[]
  cleanSystem(categories: string[]): Promise<CleanupResult[]>
}
```

Categories: Windows Update Cache (SoftwareDistribution), Log files (>30 days), Error Reporting, Old Windows installations (Windows.old), Thumbnail cache, Font cache.

#### `largeFileScanner.ts`
```typescript
export interface LargeFileEntry {
  path: string;
  name: string;
  size: number;
  type: string;
  modified: string;
  extension: string;
}

export class LargeFileScanner {
  scanLargeFiles(minSizeMB?: number, path?: string, onProgress?: (pct: number, current: string) => void): Promise<LargeFileEntry[]>
}
```

Scan drives C: and D: for files >100MB. Simulated: 20-30 files (videos, ISOs, VM images, backups, large projects).

#### `duplicateFinder.ts`
```typescript
export interface DuplicateGroup {
  hash: string;
  size: number;
  count: number;
  files: Array<{ path: string; name: string; modified: string }>;
}

export class DuplicateFinder {
  scanForDuplicates(paths?: string[], onProgress?: (pct: number) => void): Promise<DuplicateGroup[]>
}
```

Simulated: 5-10 duplicate groups (downloads, backup copies, photo duplicates).

### Frontend (`src/components/CleanupView/`)

**Main (`index.tsx`):** 3-tab layout: Quick Clean | Large Files | Duplicates

**QuickClean.tsx:**
- Scan button → shows categorized list with checkboxes + size
- Quick summary: total reclaimable space
- Clean selected → progress + results
- Risk indicators: safe (green), moderate (yellow), warning (red)

**LargeFiles.tsx:**
- Configurable min size (50MB/100MB/500MB/1GB)
- Drive selector
- Scan button with progress
- Results table: name, path, size (human readable), type, last modified
- Sortable columns, filter by extension

**DuplicateFiles.tsx:**
- Path selector(s) + Scan button
- Results: grouped by hash/size, show all matching files
- Each group: file count, total wasted space, expandable file list
- Color coding: green for first occurrence, yellow for duplicates
